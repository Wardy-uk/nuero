# Vault Operating Model

SARA/NUERO treats the vault as a layered system rather than a flat dumping ground.

## Layers

Raw intake:
- `Plaud/Summaries`
- `Plaud/Transcripts`
- `Imports/`

Working notes:
- `Meetings/`
- `Calls/`
- `Projects/`
- `Areas/`
- `People/`
- `Ideas/`
- `Reflections/`

Trusted knowledge:
- `Knowledge/`

System notes:
- `Documents/System/`

## Rule

Raw capture stays raw.
SARA consolidates it into a working note in the folder where it actually belongs.

## What SARA Does

1. Ingests raw notes from Plaud and other imports.
2. Links those notes back to their source material.
3. Writes consolidated notes into relevant working folders and subfolders.
4. Promotes durable signal into `Knowledge/`.
5. Writes a daily import activity report so you can see what changed.

## Output Notes

Consolidated notes should be the note you work from.
Raw notes should be the note you fall back to when you need the original detail.
